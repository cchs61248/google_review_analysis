# Google Maps 評論分析器 - Web 版介面預覽

## 功能說明

### 1. 響應式設計
- ✅ 支援電腦瀏覽器（Chrome、Firefox、Edge、Safari）
- ✅ 支援手機瀏覽器（iOS Safari、Android Chrome）
- ✅ 自動適配不同螢幕尺寸

### 2. 主要功能區域

#### 輸入區
- Google Maps 網址輸入（支援短網址自動解析）
- 評論數量設定（1-200 則）
- 開始分析按鈕

#### 載入狀態
- 動畫旋轉載入器
- 即時進度提示文字
- 預計時間提示

#### 結果展示
- **統計卡片**：顯示爬取到的評論總數
- **分頁切換**：
  - AI 分析報告（Markdown 格式渲染）
  - 原始評論列表（含評分、內容、推薦餐點）

### 3. 設計特色

#### 色彩配置
- 主色調：Google Blue (#4285f4)
- 強調色：成功綠、警告黃、錯誤紅
- 背景：淺灰色 (#f8f9fa)
- 卡片：純白色 (#ffffff)

#### 交互效果
- 懸停效果（hover）
- 點擊反饋（active）
- 平滑過渡動畫
- 陰影層次感

#### 響應式斷點
- 桌面：> 768px
- 平板：480px - 768px
- 手機：< 480px

### 4. 技術棧

**前端**
- HTML5 語義化標籤
- CSS3 現代佈局（Flexbox、Grid）
- Vanilla JavaScript（無框架依賴）
- Font Awesome 圖示

**後端**
- Flask Web 框架
- RESTful API 設計
- CORS 跨域支援

## 使用流程

1. 打開瀏覽器訪問應用網址
2. 貼上 Google Maps 餐廳網址
3. 設定要爬取的評論數量（預設 30）
4. 點擊「開始分析」按鈕
5. 等待爬取與 AI 分析完成（約 2-3 分鐘）
6. 查看 AI 分析報告或原始評論

## 手機使用說明

### 本地開發模式
1. 確保手機與電腦連接同一 Wi-Fi
2. 查詢電腦 IP 位址：
   ```powershell
   ipconfig
   ```
3. 手機瀏覽器訪問：`http://電腦IP:5000`

### 雲端部署模式
1. 將應用部署到 Railway 或 Render
2. 獲取公開網址（例如：`https://your-app.railway.app`）
3. 手機瀏覽器直接訪問該網址

## 注意事項

1. **首次使用**：確保已設定環境變數（OPENAI_API_KEY）
2. **網路要求**：需要穩定的網路連線
3. **處理時間**：評論數量越多，處理時間越長
4. **瀏覽器支援**：建議使用最新版瀏覽器
5. **手機訪問**：建議使用 Wi-Fi 網路，避免耗費過多行動數據

## 常見問題

### Q: 為什麼手機無法訪問本地服務？
A: 確認以下事項：
- 手機與電腦在同一 Wi-Fi 網路
- 防火牆允許 5000 埠的連線
- 使用正確的電腦 IP 位址

### Q: 分析結果不完整？
A: 可能原因：
- 網路連線中斷
- API Key 額度不足
- Google Maps 頁面載入失敗

### Q: 手機瀏覽器顯示異常？
A: 建議：
- 清除瀏覽器快取
- 使用無痕模式測試
- 更新瀏覽器到最新版本
